# Loop38 视觉债务整改文档

## 目标

北极星不变：接近《智慧的再现》的明亮、温暖、可爱、圆润、玩具感，禁止回到黑暗地牢、脏绿噪点、冷硬金属和多套 HUD 混搭。

这份文档只做一件事：从玩家首局路线反推首发场景的视觉债务，并按 `P0/P1/P2` 排出整改优先级，方便后续继续 harness loop。

## 依据

我对齐了这些现有约束和命名：

- 路线：`StartCamp -> FieldWest -> FieldRuins -> DungeonHub -> DungeonRoomA/B/C -> BossArena -> 返回营地`
- 首发场景：`StartCamp`、`FieldWest`、`FieldRuins`、`DungeonHub`、`DungeonRoomA`、`DungeonRoomB`、`DungeonRoomC`、`BossArena`
- 内部测试场景：`MechanicsLab`，不进入首发主流程
- 资源/意图事实源：`asset_selection_manifest.json`、`asset_binding_manifest_v2.json`
- 风格门禁：`assets/configs/style_resource_gate.json`
- 测试约束：`tests/dungeon-scenes.test.mjs`、`tests/style-resource-gate.test.mjs`、`tests/wechat-build-config.test.mjs`、`tests/wechat-build-output.test.mjs`、`tests/first-session-journey.spec.mjs`、`tests/visual-scene-initial.spec.mjs`
- 包体门槛：微信主包 `4 * 1024 * 1024 bytes` 硬失败，`3.7MB` 预警

## 反推结论

从“玩家第一次进入并完整打一圈”反推，现在最大的问题不是单个场景的素材数量，而是三件事同时压着体验：

1. 视觉语言还没有统一成“暖、亮、圆、玩具感”。
2. 首局路线上的多个场景仍然共享了过多原型味道和暗色地牢味道。
3. HUD、场景底图、机关表现、AI 候选素材之间的边界还不够干净，导致玩家在每一段都像在看不同项目拼起来的东西。

## 边界与下一步产物

这份文档是视觉债务地图，不是最终美术裁决，也不是机器可执行测试。

它可以决定：

- 哪些场景优先收口。
- 哪些视觉问题属于 `P0/P1/P2`。
- 哪些资源方向不能再继续扩散。
- 哪些地方必须保留给人类美术拍板。

它不能决定：

- 最终地表、墙体、角色、Boss、主 HUD 的定稿图。
- AI 候选图是否可以升级为正式资源。
- 新资源是否允许进入主包。
- 某个场景是否已经达到 Gate 5 `PASS`。

下一步如果进入实现，必须先产出更细的资源决策表，至少包含：资源 key、当前状态、是否可保留、是否必须重做、是否允许进入主包、对应场景、人工验收截图、风格评分。

## P0 / P1 / P2

### P0

会直接破坏首发可发布性或首局理解的债务。

- 主包与首发资源边界必须继续收紧，不能让视觉整改把包体再撑爆。
- `StartCamp` 必须先成为“我在哪、往哪走、能点什么”的可读起点，否则后面全都在浪费。
- `DungeonHub` 必须先变成清晰的中继枢纽，否则三房间和 Boss 的主题会继续糊成一团。
- `BossArena` 不能再只是“有战斗”，必须具备战前预告、战中阶段反馈和战后回营地的视觉闭环。

### P1

会显著拖慢风格统一，但不一定立刻造成硬阻断。

- `FieldWest`、`FieldRuins` 的过暗、过噪、过原型化地表需要重做。
- `DungeonRoomA/B/C` 的主题区分度不够，三房间在 3 秒内不能稳定说出题眼。
- 现有 `common_enemy`、`boss_core`、`checkpoint`、`touch_attack_button`、`system_pause_icon` 只能当局部候选，不能直接变成全局风格锚点。

### P2

属于继续打磨和补表层的问题。

- 场景边缘装饰、远景层次、局部特效、个别道具的玩具感增强。
- 个别交互点的图标化、圆润化、柔光化。
- 某些 AI 候选小物件的二次筛选。

## 场景级债务

### `StartCamp`

问题：

- 现在更像“原型集合”，不是稳定的开局营地。
- 首屏信息层次不够：玩家很容易先注意到 HUD 和混搭边框，而不是“这是营地、这是方向、这是可交互点”。
- 底图和地标还没有形成“暖、亮、可爱”的第一印象。

推荐资源方向：

- 温暖露营地、浅米色木质构件、圆润营火、柔和草地、简化路牌、玩具感围栏。
- 地面优先低噪声、低对比、易读的浅色草地或土路。
- 营地地标要大形体、少细节、强识别。

禁止项：

- 黑灰大底、脏绿噪点、冷硬金属边框、尖锐石刺。
- 任何会让首页看起来像地牢入口的暗色阴影。
- 多套 HUD 同屏抢焦点。

验收口径：

- 5 秒内玩家能说出“我在哪、往哪走、能点什么”。
- 第一眼先看到营地，不先看到调试感或 HUD 叠层。

需要人类美术决策：

- 开局营地的最终色温锚点。
- 营地地标和主入口的最终轮廓语言。

### `FieldWest`

问题：

- 仍然偏“测试地形”而不是“温暖野外过渡带”。
- 地表和 UI 之间有明显竞争，路径边界不够干净。
- 视觉上离“明亮可爱的外野”还差一层统一的风格收口。

推荐资源方向：

- 浅暖草地、风化但不脏的石路、柔和分区边界、少量花草点缀。
- 路径和可交互区域要用颜色和形状同时表达，不靠脏噪点硬区分。
- 过渡场景应该像从营地走向远方，而不是钻进更黑的洞。

禁止项：

- 大片深色块面。
- 过度写实裂纹。
- 过密地表纹理。

验收口径：

- 进入 3 秒内能看出主路和目标方向。
- 不能出现“这是另一个更暗的地牢前庭”的错觉。

需要人类美术决策：

- 野外与遗迹的过渡比例。
- 远景、地表、路标的统一色板。

### `FieldRuins`

问题：

- 遗迹感有了，但还偏冷、偏杂、偏压抑。
- 现在更像“被噪声覆盖的灰石块”，不是“可爱的废墟冒险区”。
- 敌意感和探索感没有被很好地区分。

推荐资源方向：

- 温暖石材、浅褐破损边、圆角残墙、少量藤蔓和花草。
- 遗迹应保留历史感，但要让“可走、可读、可互动”优先于“荒凉”。
- 可使用少量古代纹样做识别点，但不能做成哥特恐怖风。

禁止项：

- 冷灰废墟主导。
- 尖刺、裂痕、压迫性红黑对比。
- 真实岩石灾难感和血腥感。

验收口径：

- 玩家能在 3 秒内看出“这是遗迹，不是洞穴”。
- 路径、入口、机关点位都要比背景更可读。

需要人类美术决策：

- 遗迹纹样该更“古代童话”还是更“神殿玩具”。
- 破损程度的上限。

### `DungeonHub`

问题：

- 这是当前最需要先收口的中继区。
- 房间入口、回营地路径、状态反馈和 HUD 叠层都在抢焦点。
- 目前的感觉更像“多个功能块堆在一起”，不是“一个清晰的冒险枢纽”。

推荐资源方向：

- 中央枢纽使用更统一的暖石大厅语言。
- 三房间入口要用强主题卡片/门扉/旗标区分。
- 回营地路径要有稳定的视觉锚点，不能和房间门长得太像。

禁止项：

- 多种边框语言混用。
- 过多程序化标签式按钮。
- 中央大厅黑化，导致所有入口都像地牢门。

验收口径：

- 3 秒内能分清“去房间、回营地、当前进度”。
- 玩家不会把 Hub 误认成普通战斗房。

需要人类美术决策：

- 枢纽大厅到底是“温暖神殿”还是“营地大厅”。
- 三房间入口的主题编码方式。

### `DungeonRoomA`

问题：

- 当前主题表现不够明确，房间识别度主要靠文字和逻辑，不靠画面。
- 画面仍然偏暗偏重，箱子题眼没有足够的“玩具感”。

推荐资源方向：

- 以箱子、木板、软垫、轻微压机关、圆润提示牌为主。
- 房间底色要保持暖中性色，不要继续用黑石块压场。
- 机关应该像“摆玩具”，不是“拆地牢”。

禁止项：

- 巨大黑石块当主视觉。
- 真实感裂石和重金属陷阱。

验收口径：

- 3 秒内能说出“这是箱子房”。
- 箱子是主角，不是背景装饰。

需要人类美术决策：

- 箱子的最终形状和情绪。
- 该不该保留一点夸张的玩具比例。

### `DungeonRoomB`

问题：

- 主题识别仍然不稳，弹花/跳跃/位移的乐趣没有被画面先说出来。
- 视觉元素容易被地板和墙体吞掉。

推荐资源方向：

- 软弹花、弹性垫、圆润导向箭头、轻快色块。
- 用更明亮的跳跃提示去讲“这关是在玩跳跃”。
- 保持房间整体比 RoomA 更轻快，但仍要统一在同一套可爱冒险语系里。

禁止项：

- 太多深红、暗紫、黑褐的压迫感。
- 让玩家看不出可交互点。

验收口径：

- 3 秒内能说出“这是弹花/跳跃房”。
- 画面要主动提示移动和节奏。

需要人类美术决策：

- 弹花该偏自然花朵还是偏机关花朵。
- 房间的节奏感要更可爱还是更强调技巧。

### `DungeonRoomC`

问题：

- 当前是三房间里最容易被噪声淹没的一间。
- 炸虫/爆破/清障的题眼没有成为主视觉。

推荐资源方向：

- 圆滚滚的爆虫、可爱危险标识、短时爆闪、清晰的危险半径提示。
- 让“炸”更像卡通机关，不像阴森爆炸。
- 视觉上要给玩家明确的预警和反馈。

禁止项：

- 过度黑紫、过度脏绿、真实爆破火焰。
- 让炸虫像恐怖昆虫或寄生体。

验收口径：

- 3 秒内能说出“这是炸虫房”。
- 爆破预告、触发和结算都要看得懂。

需要人类美术决策：

- 炸虫的可爱程度上限。
- 危险提示是“萌化”还是“强调危险但不吓人”。

### `BossArena`

问题：

- 现在的战斗结构已经能跑，但视觉上还没有完全从地牢气质里脱身。
- Boss 主体、场地和 HUD 的边界还不够干净。
- 胜利后回营地的出口感需要更明确。

推荐资源方向：

- 更像仪式场/守门台/温暖祭坛，而不是黑色深渊竞技场。
- Boss 主体应保留辨识度，但不要压过整体可爱风。
- 战前预告、阶段变化、胜利出口要有明显的颜色和轮廓变化。

禁止项：

- 黑红压迫感。
- Boss 主体写实恐怖化。
- 让护盾、伤害、胜利反馈混成一团。

验收口径：

- 战前一眼知道要打谁。
- 战中一眼知道现在处于什么阶段。
- 战后能清楚知道已经通关并可回营地。

需要人类美术决策：

- Boss 的最终轮廓和情绪。
- 胜利反馈是“庆典感”还是“温和的完成感”。

## 推荐资源方向总表

优先重做或重定调的方向：

- `outdoor_wall_standard`
- `outdoor_wall_broken`
- `outdoor_wall_cracked`
- `outdoor_path_cobble`
- `outdoor_ground_flowers`

可保留但不能当最终风格锚点的方向：

- `outdoor_ground_green`
- `outdoor_ground_ruins`

仅能局部点缀、不能升级为全局风格锚点的候选：

- `common_enemy`
- `boss_core`
- `checkpoint`
- `touch_attack_button`
- `system_pause_icon`

## 禁止项

- 黑灰大底、脏绿噪点、写实裂痕、尖锐哥特、血腥压迫。
- 冷硬金属、过重金边、过度写实的废墟质感。
- 像素、手绘、写实、AI 质感混搭成同一层视觉主语。
- 多套 HUD 同屏抢戏。
- AI 图直接定稿地表、墙体、主 HUD、角色主体、Boss 主体、关键交互提示。

## 验收口径

### 全局

- 首发主包仍必须满足微信 `4 * 1024 * 1024 bytes` 硬门槛，`3.7MB` 为预警线。
- 首发主流程必须仍然只围绕 `StartCamp -> FieldWest -> FieldRuins -> DungeonHub -> DungeonRoomA/B/C -> BossArena -> 返回营地`。
- `MechanicsLab` 只做内部测试，不进入首发视觉叙事。
- 视觉回归不能只靠“看起来更好”，必须有截图和人工评分。

### 首局

- `StartCamp`：5 秒内看懂位置、方向、可点内容。
- `FieldWest` / `FieldRuins`：3 秒内看懂主路和目标。
- `DungeonHub`：3 秒内分清房间、回营地和当前进度。
- `DungeonRoomA/B/C`：3 秒内分别说出箱子、弹花、炸虫。
- `BossArena`：看懂战前预告、战中阶段、战后回营地。

## 需要人类美术决策的点

- 主视觉到底要偏“营地童话”还是“神殿童话”，不能两边都要。
- 每个场景的最终地表、墙体、边界、远景必须有人类定稿。
- Boss 的最终轮廓、色温和阶段反馈必须有人类拍板。
- `common_enemy`、`boss_core`、`checkpoint` 这类候选能不能升级为最终资产，不能交给自动绑定自己决定。
- 主 HUD 的最终语言只能有一套，不能让程序皮肤和场景皮肤继续争夺主权。

## 结论

当前不是“素材不够”，而是“视觉秩序还没收口”。如果继续沿着可爱风走，最先该做的不是再堆新图，而是先把 `StartCamp`、`DungeonHub`、`FieldWest`、`FieldRuins` 的色温和轮廓语言统一，再把 `DungeonRoomA/B/C` 和 `BossArena` 的题眼做清楚。

下一轮 loop 最值得盯的不是“再多做几个资源”，而是“哪些资源已经可以被明确判定为最终风格锚点，哪些必须继续留在候选层”。
