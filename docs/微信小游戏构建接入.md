# 微信小游戏构建接入

## 当前接入内容

- 已提供微信小游戏命令行构建配置生成脚本：
  - `node ./tools/generate-wechat-build-config.mjs`
- 已提供一键构建脚本：
  - `node ./tools/run-wechat-build.mjs`
- 已提供构建产物校验脚本：
  - `node ./tools/verify-wechat-build-output.mjs`

## npm 命令

```powershell
npm run build:wechat:config
npm run build:wechat
npm run verify:wechat
npm run reload:wechat
npm run rebuild:wechat
npm run open:wechat
```

## 环境变量

- `COCOS_CREATOR_EXE`
  - 可选。显式指定 `CocosCreator.exe` 路径。
- `WECHATGAME_APPID`
  - 可选。覆盖默认微信小游戏 `appid`。
- `WECHATGAME_DEBUG`
  - 可选。默认开启调试构建；传 `0` 可切换为非调试构建。
- `WECHAT_DEVTOOLS_CLI`
  - 可选。显式指定微信开发者工具 `cli.bat` 路径。
- `WECHAT_DEVTOOLS_PORT`
  - 可选。显式指定微信开发者工具 CLI 端口，默认 `9420`。
- `WECHAT_DEVTOOLS_FORCE_REOPEN`
  - 可选。传 `1` 时，`open:wechat` / `reload:wechat` 会先 `quit` 再 `open`，只在 DevTools 明显缓存旧 runtime 时使用。
- `WECHAT_DEVTOOLS_SOFT_CLOSE`
  - 可选。与 `WECHAT_DEVTOOLS_FORCE_REOPEN=1` 搭配时，改为先 `close --project` 再 `open`，避免整应用重启。
- `WECHAT_REBUILD_SKIP_CLOSE`
  - 可选。传 `1` 时，`rebuild:wechat` 跳过构建前的 `close --project`。
- `WECHAT_REBUILD_SKIP_VERIFY`
  - 可选。传 `1` 时，`rebuild:wechat` 只做 `build -> open`，跳过 `verify:wechat`。

## 当前默认值

- 构建平台：`wechatgame`
- 输出目录：`build/wechatgame`
- 启动场景：`StartCamp`
- 默认 `appid`：`wx2a215f964be2b668`
- 默认朝向：`landscape`
- 关键产物：`game.js`、`game.json`、`project.config.json`、`application.<hash>.js`

## 推荐日常流

- 日常刷新微信包时，优先运行 `npm run rebuild:wechat`。
- 这条链路会先对当前项目做一次 DevTools CLI `close --project`，再执行 `build:wechat -> verify:wechat -> open:wechat`，尽量复用已打开的 DevTools 而不走鼠标自动化。
- 如果只是把最新一次成功构建重新拉到 DevTools，运行 `npm run reload:wechat` 或 `npm run open:wechat` 即可。
- GUI smoke / touch harness 仍然只用于证据采集和疑难诊断，不作为日常重建入口，因为它会碰桌面窗口和鼠标。

## 说明

- 该接入优先保证“本地可构建 + 产物可校验”。
- 首次使用微信开发者工具时，仍可能需要在 GUI 中完成登录或信任项目，这一步无法完全命令行替代。
- 当本机装好微信开发者工具后，可直接导入 `build/wechatgame` 目录继续验证，或运行 `npm run open:wechat` / `npm run reload:wechat` 通过 DevTools CLI 打开最新一次成功构建。
- 如果 `build/wechatgame` 被 DevTools 占用，构建脚本可能回退到 `build/wechatgame-staging`；`open:wechat` / `reload:wechat` 会优先跟随 `temp/wechat-build-status.json` 指向最近一次成功输出目录。
- 日常不要为了刷新构建结果去跑 GUI dismiss 脚本。那条链路主要服务于 harness 取证，会抢窗口焦点；日常入口已经切到 CLI `open/reload`。
- 如果开发者工具能打开项目，但提示当前账号不是该小程序/小游戏的开发者，优先检查 `project.config.json` 里的 `appid`。
- 默认构建会使用测试 `appid` `wx2a215f964be2b668`。如果当前登录账号没有这个 AppID 的开发权限，请在构建前设置 `WECHATGAME_APPID=<你的小游戏AppID>`，然后重新执行 `npm run build:wechat`。
