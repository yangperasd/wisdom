# 微信小游戏构建接入

## 当前接入内容

- 已提供微信小游戏命令行构建配置生成脚本：
  - `node ./tools/generate-wechat-build-config.mjs`
- 已提供一键构建脚本：
  - `node ./tools/run-wechat-build.mjs`
- 已提供构建产物校验脚本：
  - `node ./tools/verify-wechat-build-output.mjs`

## npm 命令

```powershell
npm run build:wechat:config
npm run build:wechat
npm run verify:wechat
npm run open:wechat
```

## 环境变量

- `COCOS_CREATOR_EXE`
  - 可选。显式指定 `CocosCreator.exe` 路径。
- `WECHATGAME_APPID`
  - 可选。覆盖默认微信小游戏 `appid`。
- `WECHATGAME_DEBUG`
  - 可选。默认开启调试构建；传 `0` 可切换为非调试构建。
- `WECHAT_DEVTOOLS_CLI`
  - 可选。显式指定微信开发者工具 `cli.bat` 路径。
- `WECHAT_DEVTOOLS_PORT`
  - 可选。显式指定微信开发者工具 CLI 端口，默认 `9420`。

## 当前默认值

- 构建平台：`wechatgame`
- 输出目录：`build/wechatgame`
- 启动场景：`StartCamp`
- 默认 `appid`：`wx6ac3f5090a6b99c5`
- 默认朝向：`landscape`
- 关键产物：`game.js`、`game.json`、`project.config.json`、`application.<hash>.js`

## 说明

- 该接入优先保证“本地可构建 + 产物可校验”。
- 首次使用微信开发者工具时，仍可能需要在 GUI 中完成登录或信任项目，这一步无法完全命令行替代。
- 当本机装好微信开发者工具后，可直接导入 `build/wechatgame` 目录继续验证，或运行 `npm run open:wechat` 尝试自动打开。
- 如果开发者工具能打开项目，但提示当前账号不是该小程序/小游戏的开发者，优先检查 `project.config.json` 里的 `appid`。
- 默认构建会使用测试 `appid` `wx6ac3f5090a6b99c5`。如果当前登录账号没有这个 AppID 的开发权限，请在构建前设置 `WECHATGAME_APPID=<你的小游戏AppID>`，然后重新执行 `npm run build:wechat`。
