### 14.4 mobile 触发条件迭代过程（坦白技术失败）

## 步骤 15：WeChat runtime gameplay liveness — **实证成功**（2026-04-15 晚）

### 15.1 起因

### 15.2 解法 — `runtime-smoke-when-focused.ps1`

```ps1
while ($attempts -lt 30) {

  Start-Sleep -Milliseconds 500
}
# 用户在 15 秒内点 simulator 面板，脚本自动检测到，开跑 SendInput
```

### 16.1 user 直击的 bug

user 发了一张 StartCamp runtime 截图 + 一句话 ——
> "不觉得召唤、攻击在重叠吗，这不符合美学吧，你刚才做的美学 agent 没查出这个问题？"

一眼看图就能指出。实测布局数值：
- ATTACK 按钮 118×70，位置 `(safeRight - 118, safeBottom + 116)`
- SUMMON 按钮 124×66，位置 `(safeRight - 118 - 144, safeBottom + 116 - 48)`
- 垂直方向 edge gap = `48 − (70/2 + 66/2) = 48 − 68 = **−20 px**`

硬冲突。2 个按钮在桌面分辨率下重叠 20 像素。

### 16.2 美学 agent 为什么没查出来

我之前开了 3 个并行美学 agent（A/B/C），每个的 prompt 只写了 "图标太小、覆盖、重叠、丑等角度"。这种**主观描述式 prompt**让 agent 感官上判断，但**几何上的 20px 重叠看整体截图时看不太出来**（ATTACK / SUMMON 都在"右下"，人眼判断是"挨着"而不是"叠着"）。

根因：prompt 没要求 agent **真的去算按钮坐标**。

### 16.3 三管齐下修复

**1. 修布局参数**（`GameHud.ts:applyResponsiveLayout`）：
- 把 SUMMON 的 Y 改成和 ATTACK 一样（删掉 `-48` 垂直偏移）
- 横向间距按每档 scale 精算：mobile 200 / tight 150 / compact 160 / desktop 170
- 算法：`summonOffsetX = ceil(halfW_attack * scale + halfW_summon * scale + margin)` 确保 edge_gap ≥ 20

**2. 加 runtime AABB 自检**（`GameHud.ts:verifyTouchButtonGeometry`）：
- 首次 applyResponsiveLayout 后遍历所有 active touch 节点
- 两两算 AABB 交集
- 如果重叠 → `console.warn` 打印 `dx / dy / scale`
- static `geometrySelfCheckDone` 防止每次 resize 都重刷

**3. 加 playwright 回归守卫**（`tests/hud-geometry-no-overlap.spec.mjs`）：
- 3 viewport × 2 scene = 6 个 case
- 每个 case 从 preview 里拉 world-space AABB，两两算交集
- 重叠则测试 fail，附详细 dx/dy 信息
- 防止下次类似 bug 悄悄溜回来

回归结果：
- `npx playwright test tests/hud-geometry-no-overlap.spec.mjs` — **6/6 pass**
- `npm run test:typecheck` 干净
- `npm run test:node` 162/162
- `npm run test:responsive` 15/15

### 16.4 同步做的 RectVisual 立体感升级（原 user 提的"大方块"问题）

同一轮里，user 还说 "先不说 UI 素材，你不觉得有些就是一个大方块吗，这符合美学吗"。回应：代码级升级 `RectVisual.ts`，不靠素材：

- **新增 `gradientStrength` 属性**（默认 0.35）：在 fill 之上叠 4 条等高带，上 2 条比原色亮、下 2 条比原色暗，低 alpha 叠加，生成自然 vertical gradient。corner overlap 由低 alpha 隐藏
- **新增 `innerHighlight` 属性**（默认 true）：在 rect 顶部 inset 2px 画一条比 fillColor 亮 60 的 2-px 线，作"顶光"效果
- **新增 `hatchStrength` 属性**（默认 0）：对 >= 120×80 的大 rect，可额外画**对角斜纹** —— 低 alpha（~15）稀疏 1px 线，人眼感知不到单线但能感知到 *grain*，彻底破坏"大方块"的平摊感

8 个场景的 27 个大 placeholder 区（Backdrop/Lane/Zone/Strip）全部启用 hatch (strength 0.55, spacing 22) + 提升 gradient 至 0.35。PIL 像素扫描证实 CampLeftLane 区 R 值沿对角线 0..32 波动 —— 即 hatch 线可观测。
