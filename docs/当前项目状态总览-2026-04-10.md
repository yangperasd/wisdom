# 当前项目状态总览（2026-04-10）

## 1. 目标

本项目当前的明确目标是：

- 平台：微信小游戏
- 引擎：Cocos Creator 3.8.8
- 类型：2D 俯视角动作解谜小游戏
- 目标体量：单次完整通关约 45 到 60 分钟
- 核心体验：3 个回响、轻战斗、2 段野外、3 房间迷宫、1 个 Boss

设计原始方案见：

- [方案A-MVP-微信小游戏-类智慧的再现.md](/E:/cv5/wisdom/docs/方案A-MVP-微信小游戏-类智慧的再现.md)

当前的开发方向不是复刻原作全部系统，而是做一个“能完整跑通、能打包到微信小游戏、能继续接正式素材”的竖切 MVP。

## 3. 调试方法

### 3.1 代码与场景结构调试

先跑最稳的两层：

```powershell
npm run test:typecheck
npm run test:node
```

这两层主要用于确认：

- TypeScript 编译无误
- 场景结构没被破坏
- prefab 关键组件还在
- 纯逻辑 helper 没回归

### 3.2 本地预览交互调试

如果 Cocos 预览服务已经启动，再跑：

```powershell
npm run test:smoke:local
```

或直接跑整套：

```powershell
npm run test:all
```

这层主要覆盖：

- `MechanicsLab` 预览是否能正常进入
- 摇杆移动
- 攻击按钮
- 回响切换和放置
- 压板开门
- 炸墙

对应用例：

- [preview-smoke.spec.mjs](/E:/cv5/wisdom/tests/preview-smoke.spec.mjs)

### 3.3 Cocos 编辑器内调试

编辑器内排查优先看：

- 场景 Console
- [project.log](/E:/cv5/wisdom/temp/logs/project.log)

当前常见判断经验：

- 场景里全是色块，但没有红错：
  先判断是不是 fallback 占位，不要直接认定素材失效
- `Missing class` 或 `Script ... missing or invalid`：
  优先怀疑脚本索引或场景导入缓存，而不是先怀疑玩法脚本本身
- 场景表现不更新：
  先切到别的场景再切回来，必要时重开 Creator

### 3.4 微信开发者工具内调试

当前微信开发者工具里可能出现一部分非阻塞 `warn`，常见包括：

- `SharedArrayBuffer ...`
- `getSystemInfo API 提示`
- 首次启动时 `cacheList.json` 不存在

这些在“能进画面、无持续红错”的前提下，优先视为开发者工具或引擎层噪音，不作为阻塞错误处理。

真正要优先排查的是：

- 黑屏
- 一直卡在加载页
- 无法进入 `StartCamp`
- 触屏输入无效
- 场景切换或存档链路异常

## 4. 部署方法

### 4.1 本地开发部署

当前的“部署”实际上是指把本地项目部署到微信小游戏开发环境进行运行验证。

标准流程是：

1. 在 Cocos Creator 中维护项目和场景
2. 生成微信小游戏构建产物
3. 打开微信开发者工具
4. 导入构建目录
5. 在模拟器或真机中验证

相关脚本：

- [run-wechat-build.mjs](/E:/cv5/wisdom/tools/run-wechat-build.mjs)
- [open-wechat-devtools.mjs](/E:/cv5/wisdom/tools/open-wechat-devtools.mjs)
- [verify-wechat-build-output.mjs](/E:/cv5/wisdom/tools/verify-wechat-build-output.mjs)

### 4.2 当前部署边界

当前已经打通的是：

- 本地构建微信小游戏包
- 本地打开微信开发者工具
- 在开发者工具模拟器里运行

当前还没有自动化的是：

- 微信后台上传

- 正式上线

也就是说，项目现在是“开发部署链路可用”，还不是“正式发布流水线自动化”。

## 5. 打包方法

### 5.1 标准命令

当前标准打包命令是：

```powershell
npm run build:wechat
npm run verify:wechat
npm run open:wechat
```

### 5.2 打包行为

打包脚本会依次做这些事：

1. 生成微信小游戏构建配置
2. 调用 `Cocos Creator` 执行构建
3. 对构建产物执行运行时设置优化
4. 校验产物结构、AppID、设置和包体大小
5. 打开微信开发者工具

关键脚本：

- [wechat-build-utils.mjs](/E:/cv5/wisdom/tools/wechat-build-utils.mjs)
- [run-wechat-build.mjs](/E:/cv5/wisdom/tools/run-wechat-build.mjs)
- [optimize-wechat-build-output.mjs](/E:/cv5/wisdom/tools/optimize-wechat-build-output.mjs)
- [verify-wechat-build-output.mjs](/E:/cv5/wisdom/tools/verify-wechat-build-output.mjs)

```powershell

npm run build:wechat
npm run verify:wechat
```

当前打包后建议至少确认：

- 微信开发者工具能正常打开项目
- 模拟器能进入 `StartCamp`
- 主要按钮和摇杆输入可用
- 没有黑屏和持续红错
- 输出目录总包体没有超过当前预算

## 6. 当前已知问题与注意事项

- 编辑器场景视图里仍可能看到部分色块或标签，这是当前“素材接入过渡态”的一部分，不代表项目逻辑失效
- 如果 Creator 打开场景时报一批 `missing class`，优先怀疑启动时脚本索引和场景导入顺序，而不是直接怀疑脚本源码
- 微信开发者工具中的部分 `warn` 来自工具或引擎层，不应与项目自己的红色运行错误混为一谈
- 当前最稳的验证顺序仍然是：
  - `npm run test:typecheck`
  - `npm run test:node`
  - 再做 Cocos 预览或微信开发者工具人工验证

## 7. 一句话结论

项目现在已经处于“主流程可跑、自动化可用、微信开发包可生成、素材接入层正在替换占位符”的阶段。

它还不是最终美术完成版，但也已经不是只有色块和空壳逻辑的原型。后续工作的重点会从“搭底座”逐步转向“清理占位、补正式表现、做真机体验收口”。
