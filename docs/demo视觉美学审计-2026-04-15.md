**本轮修复（三批合流）**：
1. **第一批**：`GameHud.ts:364` 锁态 label 从写死 `LOCK` 改成 `${buttonName}·锁`，现在看到的是"花·锁 / 炸·锁"不再重名

### 10. 🟢 已修（SUMMON 改冷钢蓝）按钮色系无语义规律

`ATTACK`（橙）/ `SUMMON`（绿）/ `BOX`（米）/ `LOCK`（灰）四色策略既不冷暖对应功能也不层级对应频率，形成"彩虹糖"式廉价感。

- 攻击/防御类 → 暖色族（红/橙）
- 回响/召唤类 → 冷色族（青/蓝）
- 状态类（锁/解锁）→ 明度区分而非色相区分

**本轮修复**：`TouchPlaceEcho-Visual`（SUMMON 按钮）fillColor 从 `(131,207,145,245)` 暖绿改为 `(72,134,186,245)` 冷钢蓝，stroke 同步调整。影响范围：全 8 个主流程场景。

现在按钮语义分三层：
- **攻击族（暖橙）**：`TouchAttack`（236,159,89）+ `TouchRespawn`（111,69,74）
- **召唤族（冷钢蓝）**：`TouchPlaceEcho`（72,134,186）← 本轮改色
- **主题色（按回响类型）**：`TouchEchoBox`（214,176,96 棕）/ `TouchEchoFlower`（67,146,88 绿）/ `TouchEchoBomb`（156,68,66 红）
- **状态辅助**：`TouchPause`（71,87,108 冷灰）/ Lock state（42,49,58）

"SUMMON 绿"和"TouchEchoFlower 绿"的撞色隐患一并消除。

### 11. 🟢 已修 三套美术语言割裂

同场景内同时出现：
- **Denzi 像素风**（角色、石墙、地砖）

## 跨视口观察（桌面 preview vs 移动端 runtime）

**同一套 HUD 在桌面 1280 宽下占屏 ~18%，在手机 350 宽下占屏 ~55%**。问题不在 UI 变大了，而是**没有断点适配** — 桌面布局 1:1 缩进手机里。

**分发前强烈建议做**：
- **Mobile HUD skin**：不是缩放桌面版，是重绘。按钮 icon 化、文字砍 70%、关键信息图形化
- **Safe-area padding 统一 16-20 px**：WeChat 圆角画布下任何 UI 都不被切
- **BossArena Boss 站位重做**：放大 1.8x + 居中 + 脚下加战斗圈 + 远景背板
- **双通栏 banner 合并**：目标提示和机制提示二选一，优先机制
- **移除 LOCK 按钮的重复**：肉眼可见的 UI 出厂 bug
