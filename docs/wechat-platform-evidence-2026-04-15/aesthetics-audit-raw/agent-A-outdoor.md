# Visual Aesthetics Audit — Agent A (Outdoor Scenes)

> Raw output from the first parallel visual aesthetics agent. Scope: 3 outdoor main-flow scenes (StartCamp / FieldWest / FieldRuins). Captured 2026-04-15 evening.

## === StartCamp-initial.png ===
- 第一印象：顶栏深色条 + 中央两大块 red/gray playfield + 右下彩色按钮堆，视觉重心被打散成 5 块，眼睛第一秒不知道看哪——最刺眼的反而是右下橙色 ATTACK，而不是角色/目标。
- P0 必修：
  1) 右侧"放置箱子压住机关""BOX TRAINING""SLIME""PRESS PLATE""先尝试移动与攻击""Goal …"五个绿色/米色胶囊标签挤成一摞，字体/字号/圆角/底色几乎一致，层级完全糊了——必须做 2-3 级视觉分层（目标主提示 > 交互标签 > 敌人铭牌）。
  2) 玩家角色（那个粉紫小像素人）被塞进一大块暗红矩形里，面积占比 < 3%，主角完全不是视觉焦点；红色矩形还抢走整张图最大面积的色彩注意力。
- P1 建议修：
  1) 顶栏"营地入口 / HP 3/3 / Echo Box 1/3 / PAUSE"四项等距平铺像 Excel 表头，缺少分组与权重——至少 HP 应该带图形化血条，而不是纯文字。
  2) 右下 BOX / LOCK / LOCK / ATTACK / SUMMON 五颗按钮三种配色（米色/灰/橙/绿），色系打架；相邻两颗 LOCK 颜色/字重完全相同等于在浪费视觉槽位。
  3) 左边红色 playfield 与右边石砖 playfield 之间一条竖直黑缝 + 两种地砖突兀拼接，没有过渡带/边框/光影。
- P2 打磨：Goal 横幅里 "Goal" 四个字母和中文之间没空格分隔线；PAUSE 在深底板上纯白无图标偏单调。
- 最大亮点：右下角胶囊按钮的圆角 + pill 造型本身是耐看的语言，如果全局统一到这套就成立。

## === FieldWest-initial.png ===
- 第一印象：信息密度最低反而最耐看的一张，但顶部 Goal 文字和"触碰回声"两个标签在同一 y 轴打架，视觉第一落点落在了中央巨大的空暗红地面，玩家角色几乎看不见。
- P0 必修：
  1) 角色（左侧蓝色小像素人）尺寸 < 20px，和屏幕正中央大片空地的"留白"对比严重失衡——要么镜头推近，要么角色放大 1.5×，否则玩家进入第一秒完全不知道自己在哪。
  2) 中央偏右的"SCOUT"米色标签孤立地挂在空中，没有任何锚点指示它标的是谁（敌人？点位？），纯粹的视觉噪点。
- P1 建议修：
  1) Goal 文字 "拉窖草药,受已突娇树拉进戏旷吗口"（OCR 读起来就是乱码观感）——字号小 + 深底黑字 + 字距过挤，手机尺寸下基本没可读性。
  2) 右侧一小条石砖地像是被截断的，和左边大红地的边界不自然，构图上右重左轻不平衡。
  3) 摇杆 + ATTACK/SUMMON 三颗按钮在这个场景里相对全场占比过大（约屏幕 1/4），因为场景信息少，按钮显得更喧宾夺主。
- P2 打磨："回声春花""触碰回声"两个中文胶囊和 SCOUT 英文胶囊字体/粗细明显不同系——统一字族。
- 最大亮点：暗红主色 + 少量蓝像素角色 + 石砖点缀，这套配色如果把角色放大、去掉漂浮标签，是有氛围的。

## === FieldRuins-initial.png ===
- 第一印象：三张里最接近"成品感"的一张——大片灰石砖底 + 居中角色 + 顶部 Goal，但右侧"UNLOCK BOMB""PLACE BOMB BU…"两个胶囊被右边缘切掉，破坏了整张图的框架完整性。
- P0 必修：
  1) 右上"PLACE BOMB BU…"被视口裁断，是任何美术评审都会当场打回的低级错误——要么缩小标签，要么改为图标化。
  2) 左中"BACK TO WEST" 黄色胶囊 紧贴角色头顶，而且颜色饱和度（亮黄）是全屏最高，直接把视线从角色+目标门（屏右）拉回左边——反向引导。
- P1 建议修：
  1) "GUARD" 米色标签漂在石砖地正中央，没有对应的敌人精灵（或精灵太小看不出来），变成一个无锚点文字。
  2) 石砖地砖重复花纹过于规整（明显 tile），缺少苔藓/裂缝/光影破平，跟"Ruins"（废墟）的主题词反差——主题没兑现。
  3) Goal 文字英文 "Unlock BombBug, blast the wall, and enter the trial gate."——长句单行挤在窄条里，断句位置不合理。
- P2 打磨：右下 BOX / LOCK / LOCK 三颗按钮和左侧深红"UNLOCK BOMB"按钮色温不统一（暖红 vs 冷米黄）。
- 最大亮点：中央石砖大面积 + 居中玩家，是三张里构图最稳的一张，调整提示标签后可以直接作为主视觉模板。

## === 3 张图的共同问题（跨场景一致性）===
1) 提示胶囊滥用：每张图都漂浮着 3-6 个同款圆角胶囊（Goal / TRAINING / SCOUT / GUARD / SLIME / PRESS PLATE / BACK TO WEST 等），形状一致但功能从"世界铭牌"到"训练脚手架"到"任务目标"全混在一起，视觉上无法区分——**必须至少做 3 档：任务(大/主色)、教学(中/描边)、铭牌(小/半透)**。
2) 顶部 "标题 | HP | Echo Box | PAUSE" 条四项均等间距、同字号、无图标，信息层级为零；Echo Box 1/3 这种核心收集状态应该图形化。
3) 摇杆压在左下屏幕边缘 (距左/下边 < 20px)，在 9:16 手机上拇指会半悬空；且摇杆灰底饱和度过低，与场景融一起，操控反馈弱。
4) ATTACK(橙) / SUMMON(绿) / BOX(米) / LOCK(灰) 按钮四色策略无语义规律（攻/召唤/拿箱/锁是四种不同性质，但颜色既不冷暖对应功能也不层级对应频率），形成"彩虹糖"式廉价感。
5) 像素风角色+像素砖地 vs 扁平圆角胶囊 UI vs 纯色占位矩形三套美术语言同框，最割裂的是**纯色矩形 playfield（StartCamp 左红块、FieldWest 大红地）**——在其他两套资产都升级前，这块看上去像是 placeholder 没删。
6) 没有一张图有明确的"该往哪走"引导（箭头、光晕、动态高亮）——全靠文字胶囊解释，而胶囊本身又没层级——认知负担集中在"读字"上，不是"看图"。
