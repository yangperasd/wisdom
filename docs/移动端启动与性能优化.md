# 移动端启动与性能优化

## 本轮目标

这一轮先处理微信小游戏里我们可以稳定控制的部分：

- 默认构建改成非 debug，减少运行时日志和调试开销
- 构建后自动修正运行时设置，确保 2D 项目不会回退到不合适的手机配置
- 在校验阶段输出包体体量，避免后续无感变胖

## 当前策略

- `npm run build:wechat`
  - 先生成构建配置
  - 再调用 `Cocos Creator` 构建微信小游戏
  - 构建完成后自动执行运行时设置优化

- `npm run optimize:wechat`
  - 只对当前构建产物再执行一次运行时设置优化
  - 适合手动改过产物后重新收口

- `npm run verify:wechat`
  - 校验产物结构
  - 校验 `appid`
  - 校验是否仍然是 2D 友好的运行时设置
  - 输出总包体字节数

## 当前优化项

- 默认 `debug = false`
  - 如需调试版构建，显式设置 `WECHATGAME_DEBUG=1`

- 运行时设置自动修正为：
  - `engine.debug = false`，除非显式开启调试
  - `physics.physicsEngine = physics-builtin`
  - `screen.exactFitScreen = false`
  - `assets.downloadMaxConcurrency <= 8`
  - `splashScreen.watermarkLocation = hidden`

## 推荐命令

```powershell
npm run build:wechat
npm run verify:wechat
```

调试版构建：

```powershell
$env:WECHATGAME_DEBUG='1'
npm run build:wechat
npm run verify:wechat
```
