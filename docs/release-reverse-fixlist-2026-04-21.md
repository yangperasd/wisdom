# 反推整改清单：可爱风微信小游戏 MVP（2026-04-21）

## P0：发布阻断

| ID | 阻断项 | 动作 | 验收 |
|---|---|---|---|
| P0-01 | 微信预算仍有旧宽松 demo 口径 | 所有活跃构建常量、测试、校验改为 `4MB` 硬失败，`3.7MB` 预警 | `wechatPackageBudgetBytes === 4 * 1024 * 1024` |
| P0-02 | 当前主包大于微信硬线 | 主包极小化；后续场景/资源拆分包或远程 | `build/wechatgame` 主包 `<= 4MB` |
| P0-03 | `verify:wechat` 未成为发布门禁 | CI 调用微信构建校验；无构建产物时必须显式跳过或由 `REQUIRE_WECHAT_VERIFY=1` 强制失败 | CI 日志出现 WeChat verify 结果 |
| P0-04 | 远程资源可能误放脚本 | 校验 `remote/` 不含 `.js`, `.mjs`, `.cjs`, `.wasm` | 远程脚本列表为空 |
| P0-05 | `dungeon-scenes` 奖励断言漂移 | 统一 `pickup_relic` 绑定和测试断言 | `tests/dungeon-scenes.test.mjs` 全绿 |
| P0-06 | 首发场景集合不清 | `MechanicsLab` 明确为内部测试，不进入首发微信主流程 | build scene list 只含 8 个首发场景 |

## P1：风格和工程收口

| ID | 问题 | 动作 | 验收 |
|---|---|---|---|
| P1-01 | 风格只靠口号 | 使用审计文档的 8 项风格评分表 | 核心场景截图总分 `>= 12/16` |
| P1-02 | 黑暗地牢风险 | 禁用黑灰大底、脏绿噪点、写实裂纹、尖锐哥特、血腥压迫 | 命中禁用项直接回炉 |
| P1-03 | AI 资产边界不硬 | 禁止 AI 图直接定稿地表、墙体、主 HUD、角色主体、Boss 主体、关键交互提示 | AI 资产只作为候选或局部点缀 |
| P1-04 | 大面积 tile 不合目标 | 重做 `outdoor_wall_standard`、`outdoor_wall_broken`、`outdoor_wall_cracked`、`outdoor_path_cobble`、`outdoor_ground_flowers` | 铺贴后低噪声、路径清楚 |
| P1-05 | HUD 多头控制 | 先锁唯一 renderer owner，再标记过渡层，最后删除重复 owner | 同一 HUD 区域无多 owner |
| P1-06 | manifest 和 runtime 漂移 | 增加 manifest 对账：设计意图、绑定结果、场景接线一致 | 新图接入必须更新 manifest |
| P1-07 | 视觉回归无硬标准 | 每个核心场景建立基线截图和人工评分记录 | 截图、评分、禁用项检查齐全 |

## P2：玩家首局产品化

| ID | 问题 | 动作 | 验收 |
|---|---|---|---|
| P2-01 | 启动等待无兜底 | 加 loading/fallback 文案和弱网提示 | 玩家不会误以为卡死 |
| P2-02 | 首屏理解不足 | 首屏 5 秒内表达当前位置、方向、可操作项 | 80% 新玩家能说出目标 |
| P2-03 | 触控教学不足 | 用触屏完成移动、攻击、召唤、暂停 | 90% 新玩家 60 秒内完成 |
| P2-04 | 移动端复活入口弱 | 死亡后提供明确复活入口 | 不依赖键盘隐藏键 |
| P2-05 | 场景切换硬 | 加轻量过场、目标保留、路径反馈 | 切场后不迷路 |
| P2-06 | 三房间辨识度弱 | A/B/C 分别强化箱子、弹花、炸虫主题 | 3 秒内说出房间主题 |
| P2-07 | Boss 收口弱 | 增加战前预告、战中阶段反馈、战后结算 | 通关后知道已完成和下一步 |
| P2-08 | 软锁风险未集中验证 | 全流程误触、死亡、返回、切场专项验收 | 软锁率为 0 |

## 包体矩阵

| 包 | 内容 | 状态 |
|---|---|---|
| 主包 | 启动壳、`StartCamp`、核心脚本、首屏必需资源 | 必须小于 `4MB` |
| 分包：field | `FieldWest`, `FieldRuins` 及对应场景资源 | Gate 1 后设计 |
| 分包：dungeon | `DungeonHub`, `DungeonRoomA/B/C` 及对应资源 | Gate 1 后设计 |
| 分包：boss | `BossArena` 与 Boss 资源 | Gate 1 后设计 |

| test-only | `MechanicsLab` 与内部测试资源 | 不进入首发主流程 |

## 风格禁用速查

| 禁用 | 处理 |
|---|---|
| 黑灰大底、脏绿噪点、写实裂纹 | 回退或重做 |
| 尖锐哥特、血腥压迫、冷硬金属 | 回退或重做 |
| 过重金边、多套 UI 混搭 | 回退到单一 HUD 语言 |
| 像素/手绘/写实/AI 质感同屏混搭 | 先定唯一风格锚点 |

## 首局验收速查

| 阶段 | 通过标准 |
|---|---|
| 冷启动 | 有反馈，无假死 |
| 首屏 | 5 秒内理解目标 |
| 基础操作 | 60 秒内完成移动、攻击、召唤、暂停 |
| 死亡 | 一次明确操作可复活 |
| 路线 | 主流程完整跑通 |
| Boss | 有明确结束和返回 |
| 全程 | 软锁率 0 |
